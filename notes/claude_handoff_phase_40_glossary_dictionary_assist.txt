Phase 40 - Dictionary-Assisted Glossary Definitions

Prompt:
prompts/prompt_40.txt

Branch:
phase-40-glossary-dictionary-assist

Implementation summary:
- Added shared Admin dictionary lookup response/suggestion types.
- Added environment-driven dictionary provider config with disabled default:
  DICTIONARY_PROVIDER=disabled or DICTIONARY_PROVIDER=merriam-webster.
- Added server-side Merriam-Webster Collegiate Dictionary API client in
  apps/api/src/services/dictionary.ts.
- Added Admin-only POST /api/admin/glossary/lookup. The route returns graceful
  lookup statuses instead of blocking manual entry on provider failures.
- Added test-mode protection so NODE_ENV=test disables provider lookups by
  default, even if local .env contains a real Merriam-Webster key.
- Extended /admin/glossary so Admins can request suggestions, apply one into
  the definition field, edit it before saving, or ignore/use manual source.
- Preserved Phase 39 source metadata behavior:
  dictionary_suggested metadata is sent only after explicit suggestion
  application; manual source clears provider metadata.
- Updated .env examples, global environment docs, and unreleased notes.

Provider notes:
- Request URL used:
  https://www.dictionaryapi.com/api/v3/references/collegiate/json/{term}?key={key}
- Response normalization prefers shortdef values, strips Merriam-Webster text
  tokens to plain text, caps returned suggestions at 6, and treats string-array
  responses as spelling suggestions/no exact match.
- Terms page currently states free non-commercial API use is limited to
  1000 queries/day/reference work. Production account/key ownership should stay
  with the client.
- Branding page currently requires Merriam-Webster branding when content is
  displayed. The Admin-only UI uses proper Merriam-Webster product naming and
  source attribution text; please review whether the production UI also needs
  an approved logo asset before enabling the provider for client use.
