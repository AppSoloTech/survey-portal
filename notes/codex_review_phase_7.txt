## Findings

1. High â€” Mobile active-survey view hides the only in-progress exit path.

   At `<=760px`, the new CSS hides `.survey-list-panel` whenever the
   dashboard wrapper has `survey-active`. That makes the runner the first
   visible content, but the active-question runner only renders `Previous` and
   `Next`; the existing `Back to surveys` control is only rendered on the
   completion panel. On mobile, a participant who starts or resumes a survey
   cannot return to the survey list from an in-progress question without
   completing the survey or reloading/navigating away.

   Affected code:
   - `apps/web/src/styles.css`, lines 1616-1618
   - `apps/web/src/pages/UserDashboard.tsx`, lines 454-492
   - completion-only `Back to surveys`: `apps/web/src/pages/UserDashboard.tsx`, lines 421-445

   This also misses the Phase 7 prompt requirement to keep a clear way back to
   the list when using a mobile-focused/hidden-list approach.

   Suggested fix:
   Render a `Back to surveys` button in the active-question `survey-actions`
   using the existing `onClose` callback. Keep it disabled while submitting,
   matching the completion panel. The existing mobile full-width action rules
   will make it usable without new state logic.
