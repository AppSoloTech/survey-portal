Phase 43 - Public/User App Shell, Route, And Modal Accessibility

Implementation summary:

- Added route accessibility management in `apps/web/src/App.tsx` for public and
  registered-user routes. Scoped routes now map to document titles, route
  changes focus `#main-content`, and a polite live region announces navigation.
- Added a skip-to-main link before repeated header navigation and gave `<main>`
  a stable `id="main-content"` plus `tabIndex={-1}`.
- Replaced repeated header-brand `h1` elements with styled brand text.
- Normalized scoped public/user page headings to a single page-level `h1` where
  low-risk, including auth pages, dashboard, settings, category list,
  anonymous directory, not-found, and a visually hidden survey-attempt heading.
- Added `apps/web/src/components/AccessibleModal.tsx`, a small portal-based
  dialog primitive with initial focus, Tab containment, Escape close, return
  focus, labelled title, optional description, body scroll lock, and background
  `inert`/`aria-hidden` handling.
- Applied `AccessibleModal` to:
  - anonymous follow-up email modal in `SurveyAttemptPage.tsx`
  - password reset success modal in `ResetPassword.tsx`
- Converted the account menu from unsupported ARIA menu roles to a disclosure
  panel with normal links/buttons, `aria-expanded`, `aria-controls`, outside
  click close, Escape close, and trigger focus return on Escape.
- Added focused source-level tests:
  - `apps/web/src/App.test.ts`
  - `apps/web/src/components/AccessibleModal.test.ts`
- Updated draft release notes, phase log, and follow-up backlog.
