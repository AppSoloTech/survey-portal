Phase 58 - Glossary Search Result To Entry Workflow

Prompt:
prompts/prompt_58.txt

Branch:
phase-58-glossary-search-result-entry-workflow

Implementation summary:
- Replaced the Phase 57 disabled "Add entry unavailable" placeholder with a
  deliberate "Start entry from search" action on question-search result cards.
- The result action uses the current trimmed search query as the editable
  prefilled create-form canonical term.
- The result action does not create a Glossary entry. It switches to the
  Entries tab, scrolls to the existing create form, and focuses the definition
  textarea so the Admin can review/edit the term, use dictionary suggestions,
  and save through the existing create flow.
- Added confirmation before replacing non-empty unsaved create-form values with
  a selected search candidate.
- Added client-side duplicate candidate detection against loaded Glossary
  canonical terms and aliases using trim + case-insensitive comparison.
- Duplicate candidates are communicated via status/toast messaging and are not
  prefilled from search. The create submit button is also disabled while the
  create-form canonical term matches a loaded canonical term or alias.
- Added temporary source context near the create form with candidate,
  assessment, page/question location, and highlighted question text.
- The source context explicitly states it is informational only and is not saved
  with the entry. It is stored only in React state and cleared on successful
  create or reset.
- Preserved dictionary lookup behavior by reusing the existing create form and
  lookup controls after prefill.
- Preserved Phase 57 search behavior: short-query guard, debounce,
  AbortController, request-id stale response guard, and Phase 56 offset-based
  highlighting are unchanged.
- Updated focused web tests for duplicate helpers, unsaved-form detection,
  source guardrails, deliberate result action wiring, existing create endpoint
  usage, and no persisted source-reference wiring.
- Updated unreleased notes and PHASE_LOG.

Confirmed scope:

- Arbitrary text selection inside a result card is not implemented.
