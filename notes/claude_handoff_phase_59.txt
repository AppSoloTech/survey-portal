# Claude Handoff — Phase 59 Participant Issue Profile Thermometer

## Summary

Phase 59 adds a participant-facing issue profile thermometer to the registered
and anonymous survey runner. The thermometer is intentionally abstract: it
represents "issue profile in progress" / "profile building" / "profile
complete," not score, severity, violation count, or percentage of violations.

Hidden-tag privacy is the main review concern. The API computes progress from a
hidden-tag-aware survey structure server-side, but participant and anonymous
payloads receive only aggregate `issueProfileProgress` values.

## Progress Semantics

- Category unit: unique hidden `tagKey`.
- Identified categories:
  - selected answer option tags
  - Other tags when `otherText` is present
  - matching text/integer value tags
- Encountered categories: tag-bearing questions reached/revealed so far.
- Fill:
  - no identified categories: `0`
  - completed with at least one identified category: `100`
  - otherwise: `min(94, round(pathProgressPercent * 0.7 + discoveryRatio * 30))`
  - `discoveryRatio = identifiedCategoryCount / max(1, encounteredCategoryCount)`
- The API returns the raw aggregate fill. The web runner uses that current value
  during normal progress, allowing the visual to decrease when saved responses
  no longer identify a category.
- On the Ready to submit screen only, the runner displays `100` when at least
  one category is identified. This is a participant display rule, not an API
  completion status change.

## Review Focus

- Confirm participant/anonymous responses do not expose hidden tag identities:
  no `tagKey`, `tagValue`, `answerTags`, `valueTags`, `otherTags`, category
  labels, or selected tag names.
- Confirm server-side aggregate computation uses hidden tags without returning
  the hidden-tag-aware survey object.
- Confirm completion fills to 100% only when at least one category was
  identified.
- Confirm answer changes from tagged to untagged responses can reduce the
  aggregate counts/fill and do not leave stale profile progress.
- Confirm the Ready to submit screen can show `100` before final submission
  while completed/no-category and in-progress/no-category states remain at `0`.
- Confirm the sticky runner header stays visible without obscuring content on
  mobile/tablet/desktop.
- Confirm the UI copy does not imply score, severity, grade, violation count,
  or "winning."
- Confirm anonymous and authenticated flows remain behaviorally aligned.
- Confirm no reporting/admin tag behavior changed.
