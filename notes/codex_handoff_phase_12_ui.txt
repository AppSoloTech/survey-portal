What changed since round 1
--------------------------
A. Round-1 findings — fixes to verify:
   - P2 (drag race): SurveyOrganizePage.handleDragEnd now early-returns on
     `!isDraft || isSubmitting`, and the sortable `disabled` prop passed to page
     cards / question rows is now `!isDraft || isSubmitting`.
   - P3 (type reset): SurveyQuestionsPage adds an effect resetting
     `newQuestionType` to "text" when `activePageId` changes.
   - P3 (locked fields): QuestionEditor now passes `disabled={isPublished}` to the
     question text input, help text input, and required checkbox.

B. Enhancements:
   - Extracted the drag-end decision logic into a pure, dependency-free helper
     `apps/web/src/components/admin/organizeDrag.ts` (`resolveDragOutcome`,
     `resolveOverPageId`, `parseEntityId`) with unit tests
     `organizeDrag.test.ts` (11 cases). `SurveyOrganizePage.handleDragEnd` now
     builds a `questionIdsByPage` map, calls `resolveDragOutcome`, and dispatches
     the returned outcome to reorderSurveyPages / reorderQuestions /
     moveQuestionToPage. The old inline math and local `resolveOverPageId` were
     removed; the `arrayMove` import was dropped (helper has its own `moveItem`).
   - "Open in Questions" on each Organize page card now pre-selects that page:
     the Link passes `state={{ pageId }}`, and SurveyQuestionsPage seeds
     `activePageId` from `location.state.pageId` in a lazy useState initializer
     (falling back to the first page if absent/invalid).

C. Manual-test round 1 (developer feedback in notes/phase_12_test_notes.txt):
   - Organize: added per-page collapse/expand (a chevron toggle in the card
     header) plus a board-level "Collapse all / Expand all" toggle, so long pages
     can be collapsed to their headers for easy page reordering. State is a
     `collapsedPageIds: Set<number>` in SurveyOrganizePage. When collapsed, the
     question list region (and its `pagedrop` droppable node) is not rendered.
   - Questions page navigation: the page pills were first reworked into a
     contained horizontally-scrolling bar; that bar would not scroll across
     screen widths (a CSS grid blowout — see D), so per the developer's call the
     pills were REPLACED with a single dropdown page navigator (PageSwitcher.tsx):
     a labeled `<select>` of all pages (each shows `order. title (count)`),
     flanked by prev/next stepper arrows and an "X of N" indicator. This also
     removed the previously-separate redundant "Jump to page" dropdown.

D. CSS:
   - Root-caused the scroll failure: `.builder-workspace` is `display: grid` with
     a single implicit `auto` column that expanded to the pills' max-content. Even
     with `min-width: 0` on the bar, the track itself blew out. Added
     `.builder-workspace { grid-template-columns: minmax(0, 1fr); }` as general
     blowout hygiene. The pill bar was then replaced by the dropdown (C), so this
     rule now just hardens the single-column layout.
   - New `.page-switcher` styles (centered flex bar) + `.page-switcher-label`,
     `-select` (flex 0 1 auto, min 220 / max 460), `-step` (arrow buttons),
     `-meta`. Removed all `.page-switcher-pill*` rules and the old mobile
     scrollbar overrides. `.organize-board-toolbar` and `.organize-collapse-toggle`
     added for the collapse UI.
