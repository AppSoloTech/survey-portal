# Phase 63 — Results Large-Data UX Handoff

Date: 2026-07-01

Prompt: `prompts/prompt_63.txt`

## Implementation Summary

- Added bounded pagination to `GET /api/surveys/:id/attempts`.
- The endpoint now defaults to `page=1&pageSize=25`, caps page size at 100,
  validates invalid pagination input with 400s, and returns:
  `{ surveyId, attempts, pagination }`.
- Pagination is applied in SQL with `limit`/`offset`; a matching filtered count
  query supplies `totalCount`, `totalPages`, and previous/next metadata.
- Date-range filters continue to apply to report summary, CSV export, and the
  attempts list.
- The Results page now loads report summary/tag catalog separately from the
  paginated attempts browser, so page changes do not refetch the summary or tag
  catalog.
- Added an accessible summary disclosure button with `aria-expanded` and
  `aria-controls`; Refresh, CSV export, and date controls remain visible when
  collapsed.
- Replaced the bottom all-attempts card list with a denser table-like attempts
  browser, page-size choices (25/50/100), visible range text, and previous/next
  controls.
- Attempt detail remains a single panel loaded by attempt id and closes when
  pagination/date/page-size changes.
- Existing review-tag editing behavior is preserved inside the selected detail
  panel.
- Post-review: adjusted the attempts browser responsive breakpoint so tablet
  widths use stacked rows before the desktop grid can clip the Review action.
