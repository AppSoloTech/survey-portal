## Implemented Scope

- Added progressive page state resolution in shared code:
  - resolves current page
  - resolves active current question
  - exposes `currentPageQuestionIds`
  - tracks revealed question IDs per page
  - handles stale off-path answers by excluding them from the active path
  - preserves `skipTargetInNormalFlow` for normal page progression
- Updated start/resume/answer responses to include `currentPageQuestionIds`.
- Reused the existing single-question answer endpoint as the progressive runner
  save path.
- Kept the page batch answer endpoint available for compatibility.
- Updated participant web runner:
  - initially renders only the first visible question on the page
  - appends the next visible question after Continue
  - keeps answered on-path questions visible and editable
  - recalculates the path after Update
  - routes immediately when a triggered jump moves to a later page
- Updated shared tests for progressive reveal, stale answer behavior,
  `JUMP_TO_PAGE`, `JUMP_TO_QUESTION`, hide rules, and page path projection.
- Updated API test fixtures so legacy jump/skip scenarios are represented as
  one-question-per-page surveys, matching migrated legacy survey behavior.

## Intended Runtime Behavior

- Pages remain the organizing unit.
- The participant runner reveals questions progressively within a page.
- A saved answer can either:
  - reveal the next visible question on the same page, or
  - route immediately to a later visible page if navigation rules trigger.
- If an earlier answer on a page is edited and now branches away, later saved
  answers on that page remain stored but disappear from the active runner path.
- `HIDE_QUESTION` rules work as progressive visibility rules; hidden questions
  are never appended.
- `JUMP_TO_PAGE` is the preferred navigation rule.
- `JUMP_TO_QUESTION` remains compatibility behavior and resolves to the page
  containing the target question.
- Same-page `JUMP_TO_QUESTION` is a page-navigation no-op.
- `skipTargetInNormalFlow` still excludes jump targets from normal progression
  unless explicitly reached by a jump.

## Non-Goals

- No new database migration.
- No new rule priority controls.
- No deletion of stale/off-path answers.
- No removal of page batch answer endpoint.
- No attempt to focus a target question after `JUMP_TO_QUESTION`; the target is
  used only to determine its containing page.
- No backward or same-page page jumps.
