## Scope

This is a targeted follow-up review for a post-Claude UX/ergonomics update on
`/admin/tags`.

The Phase 29 foundation already received a clean review in
`notes/claude_review_phase_29.txt`. After that review, the developer requested
better handling for large catalogs with hundreds of category/value pairs:

- Clicking a grouped or ungrouped tag should open a group selector to move it.
- Manual drag/drop move functionality should remain.
- Groups and the Ungrouped section should be collapsible and expandable.

Please focus this review on the latest UX delta, not a full re-review of the
database/API foundation unless the new UI behavior exposes a new concern.

## Changes Made In This Update

- Added selected-tag move state:
  - clicking a tag pair opens an inline `Move to` panel
  - the visible `Move` button opens the same panel
  - moving uses the existing `moveTagDefinition` API and appends the tag to the
    selected target section
  - opening edit mode closes the move panel for that row
- Preserved drag/drop behavior:
  - drag handles still use the existing sortable row behavior
  - tag pair clicks are separate from the drag handle
  - collapsed sections still provide a droppable target so drag moves remain
    possible
- Added section collapse state:
  - Ungrouped can collapse/expand
  - each group can collapse/expand
  - board-level `Collapse all` / `Expand all` controls affect all sections
  - collapsed sections show a compact count and drop hint
- Added CSS for:
  - clickable tag pair button
  - inline move panel
  - row shell
  - collapsed drop zone
- Updated docs:
  - `markdown/FOLLOW_UPS.md` manual checklist now includes click-to-move and
    collapse/expand checks
  - `markdown/PHASE_LOG.md` records this as a post-review UX polish and notes
    that it was not part of the original Claude review

## Review Requests

Please review:

- Whether click-to-move and drag-to-move can coexist without confusing pointer
  or keyboard interaction.
- Whether collapsed sections still behave as valid drag/drop targets.
- Whether the move panel handles same-group selection, submit disabling, and
  stale state safely.
- Whether collapsed/expanded state has any bad interaction with editing,
  deleting, or moving tags.
- Whether the added controls remain usable for catalogs with hundreds of tag
  pairs.
- Whether the CSS introduces layout issues on narrow widths or dense rows.
- Whether any additional focused tests are warranted before commit.
