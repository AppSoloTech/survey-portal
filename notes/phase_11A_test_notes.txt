Survey Flow Map

Finding:
- The flow map did not account for page-scoped question numbers. A survey with
  Page 1 Question 1, Page 1 Question 2, Page 1 Question 3, and Page 2 Question 1
  was displayed as if the two "Question 1" items were globally sequential.
- This produced false "unreachable question" warnings for Page 1 Questions 2
  and 3.
- Screenshot reference: `screenshot/flow_1.png`

Resolution:
- Fixed shared `resolveNextQuestion` so normal advancement preserves the
  page-aware order from `getOrderedQuestions` instead of re-sorting candidates
  by page-scoped question display order.
- Updated the admin flow graph to use `getOrderedQuestions` and page-aware
  labels.
- Flow map nodes now show a global flow sequence marker plus `Page N` and
  `Question N` badges.
- Added shared and web regression tests for the duplicate page-scoped
  question-number case.

Validation:
- `npm run build -w packages/shared`
- `npm run test -w packages/shared`
- `npm run test -w apps/web`
- `npm run typecheck`

Logic rules
1) Source questions dropdown orders (Target question dropdown and others as well) questions by question number. To have clear and meaningful distinctions between question numbers the belong to a page, normalization format such as P1-Q1 to represent page 1 question 1.
2) Right now, if there are more than 1 skip rules on a page, the rule with the lowest question number (or it could be the first rule triggered) gets applied and the remaining become non-funtional.
3) Brainstorm and consider a reconsiliation of current skip tooling to reduce redundency and improve user usability. Right now, certain paradigms don't work due to now rendering multiple questions per page so a triggered skip logic on question 1 to jump to page 3 question 2 does not quite work as intended because the user will still see question 1.

Resolution for item 1:
- Added a shared admin label format for question references: `P{page}-Q{question}`.
- Updated logic rule source question dropdowns, target question dropdowns, skip target checkboxes, existing rule editors, and grouped rule headings to use `P1-Q1 Question text` style labels.
- Updated the Questions page editor/preview labels and related question toasts/confirmations to use the same locator format where page context matters.
- Kept page dropdowns as page labels, since those are already page-scoped.

Resolution for items 2 and 3:
- Page navigation rules now reconcile at page-submit time instead of stopping at
  the first triggered source question.
- All matching `JUMP_TO_PAGE` and `JUMP_TO_QUESTION` rules from visible
  questions on the submitted page are evaluated together.
- `JUMP_TO_QUESTION` remains supported, but it behaves as a jump to the page
  containing that question; same-page question jumps remain no-ops.
- When multiple navigation rules trigger on one page, the farthest valid later
  visible page wins.
- `HIDE_QUESTION` rules continue to union together, so multiple matching hide
  rules from one page can all hide their targets.
- The add-rule UI now defaults to `Jump to page`, keeps `Jump to question
  (legacy)` editable, and explains the farthest-page-wins behavior in the flow
  map notes.
