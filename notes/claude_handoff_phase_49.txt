## Phase Scope

Implemented:

- Added migration `database/migrations/0037_performance_test_runs.sql`.
- Added shared performance test run status, summary, detail, list, and detail-response types.
- Added Admin-only read endpoints:
  - `GET /api/admin/performance-runs`
  - `GET /api/admin/performance-runs/:id`
- Added newest-first pagination for the list endpoint.
- Added positive integer validation for route id and pagination query parameters.
- Added focused Admin API tests for authorization, pagination, shape, detail, 404, invalid params, and POST non-availability.
- Added migration-test coverage for the new table.
- Updated `markdown/DATA_MODEL_VISION.md`, `markdown/releases/unreleased.md`, and `markdown/PHASE_LOG.md`.

Non-goals:

- No `loadtest/` CLI harness.
- No k6 scenarios.
- No direct-DB load tests.
- No Azure metric sampling.
- No Admin UI pages.
- No browser "Run test", "Stop test", or "Schedule test" controls.
- No App-Service-hosted load generation.
- No Azure Container Apps, Container Instances, Azure Load Testing, queues, functions, Blob Storage, or new cloud dependencies.
- No `performance_test_samples` time-series table.
- No public or participant-facing APIs.

Known tradeoffs:

- The list endpoint selects `config`, `summary`, and `report_markdown` from SQL because it reuses the same DB row mapper, but the list response deliberately maps only summary fields and does not expose those detail-only fields.
- `config` and `summary` are typed as `Record<string, unknown>` because Phase 50 CLI payload shape may evolve; core list/query fields remain relational columns.
- Invalid pagination values return 400 instead of silently falling back. The existing Admin users list has some fallback behavior, but Phase 49 explicitly called for validating pagination parameters.
- There is no database trigger for `updated_at`; Phase 50 direct DB persistence should set `updated_at = now()` on row updates, matching existing app update patterns.

Deferred work:

- Phase 50: CLI load-test harness and direct PostgreSQL persistence.
- Phase 51: read-only Admin UI report viewer.
- Optional time-series sample table remains deferred until UI/report needs prove it is necessary.
