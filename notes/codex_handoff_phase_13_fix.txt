The finding (P2)
----------------
surveyFlowGraph added JUMP_TO_PAGE branch-page questions to the conditional-only
badge set, but node.normalNextQuestionId and the reachability adjacency still
came from the shared resolveNextQuestion(), which only excludes JUMP_TO_QUESTION
targets from question-level normal flow. So a branch page could be badged
"Conditional only" while its normal-flow arrow/reachability still walked into
another skipped branch page (e.g. Engineering -> Sales), and an unfirable
branch-page jump could be hidden because normal adjacency still reached the page.

The fix (apps/web/src/components/admin/surveyFlowGraph.ts)
---------------------------------------------------------
- Stopped using resolveNextQuestion() in the graph (import removed). Added a
  graph-local resolveGraphNormalNext() that mirrors the runtime page navigation
  (getNextVisiblePage + getNormalFlowExcludedPageIds/QuestionIds):
    * skips JUMP_TO_QUESTION skip-target questions (question-level), AND
    * skips questions on a skipped branch page (page-level) UNLESS the candidate
      shares the from-question's page — so a participant who jumped onto a branch
      page still walks the rest of that page (q2 -> q3) before leaving it.
- Compute normalNextByQuestionId once for all questions and feed it to BOTH
  node.normalNextQuestionId and buildAdjacency (buildAdjacency no longer takes
  `survey`; it takes the precomputed map). Reachability now matches the runtime:
  a branch page is reachable only via its firable jump edge, so an unfirable
  page jump correctly yields an "unreachable" branch page.
- The conditional-only badge logic is unchanged in effect; the two excluded sets
  (normalFlowExcludedPageIds, normalFlowExcludedQuestionIds) are now named to
  match the shared helpers and reused for both the badge and normal-next.
