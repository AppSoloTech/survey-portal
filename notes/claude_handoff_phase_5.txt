## Implementation Details

### Base-User Text State Isolation

- Replaced single shared `answerText`, `answerInteger`, and `selectedAnswerOptionIds` state with question-id keyed maps.
- Hydrates those maps from saved attempt responses while preserving unrelated draft answers.
- Uses the active question id to read and write the current input value.
- Keys the active question form and text/integer controls by question id to prevent React input reuse from carrying one text question's value into another.

Review focus:

- Confirm a conditional path from text Q1 -> selection Q2 -> text Q5 shows Q5 empty unless Q5 has its own saved answer.
- Confirm going backward and choosing a non-skip path does not clear unrelated saved responses.
- Confirm resume/back/complete behavior from Phase 3 is preserved.

### Admin Builder UX

- Metadata save no longer edits survey status. It preserves the current status and reports `Survey metadata saved`.
- Added a top status action panel and a bottom `Finish construction` panel showing current status plus `Publish survey` and `Retire survey` actions.
- Publish/retire still call the existing backend status endpoint, preserving backend validation and validation messages.
- All builder mutations now set scoped success feedback:
  - survey create
  - metadata save
  - question add/save/delete/reorder
  - option add/save/delete/reorder
  - hidden tag add/save/delete
  - jump rule add/save/delete
- Add-rule form clears `sourceQuestionId`, resets the form, and leaves saved rules intact after a successful create.
- Source answer and target question selects are keyed by source question to avoid stale dependent choices.
- Option and tag rows now have clearer labels and scoped buttons such as `Save option`, `Delete option`, and `Remove tag`.
- CSS improves section separation, status panel hierarchy, option/tag row grouping, button wrapping, and mobile stacking.

Review focus:

- Confirm the builder remains a dense operational tool, not a marketing/visual-designer UI.
- Confirm no decorative nested-card pattern was introduced beyond compact repeated item rows.
- Confirm labels make save/add/delete scope clear.
- Confirm mobile width does not overflow with long labels/buttons.
- Confirm backend publish validation messages still display when publishing is blocked.
