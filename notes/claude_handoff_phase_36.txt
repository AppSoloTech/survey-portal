# Claude Handoff — Phase 36 Published Survey Hidden Tag Maintenance

## Summary

Phase 36 adds a narrow admin-only exception to the published survey structural
lock. Admins can now maintain hidden tag metadata on draft and published
surveys, while retired/deleted surveys and all structural survey edits remain
locked.

## Implemented

- Added `prompts/prompt_36.txt`.
- Added a tag-specific backend lock guard in
  `apps/api/src/routes/surveyBuilderRoutes.ts`.
- Switched answer-option tag, Other tag, and value-tag routes from the
  structural draft-only guard to the new tag metadata guard.
- Added `PUT /api/surveys/:id/questions/:questionId/value-tags/:valueTagId`.
- Added the matching web API helper and admin Questions UI edit forms for value
  tags.
- Updated the admin published-survey banner copy to explain that structure is
  locked while hidden tags can be maintained for reporting.
- Kept retired surveys locked for tag metadata.
- Updated `markdown/releases/unreleased.md`.

## Important Behavior

- Published surveys still reject question, page, option text/order, template,
  and conditional-rule structural changes.
- Published surveys allow admin hidden-tag add/update/delete for:
  - answer-option hidden tags;
  - Other hidden tags;
  - text/integer value tags.
- Reporting propagation is metadata-driven: reports, attempt detail, and CSV
  exports read current tag metadata and do not mutate participant answer rows.
- Participant and anonymous participant payloads still omit hidden tags.
