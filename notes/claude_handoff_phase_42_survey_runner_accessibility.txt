Phase 42 - Survey Runner Critical Accessibility

Implementation summary:

- Replaced the participant scale range input with styled native radio options.
  The selected answer still flows through the existing selected answer option id
  state and save payload.
- Removed the now-dead range slider CSS after Claude review and renamed the
  remaining selected-value scale classes to `scale-answer-*`.
- Added question-scoped runner error state so required scale validation,
  invalid integer validation, and save failures can render next to the affected
  question with role="alert" and aria-invalid.
- Added stable ids for question prompt, help text, helper text, controls, and
  error messages.
- Connected text, integer, scale, normal selection, and Other controls to the
  prompt plus relevant help/error/helper text.
- Updated integer stepper button names to include the question text.
- Replaced the aria-hidden visual-only progress track with a native progress
  element, visible current-path page context, and aria-valuetext.
- Added live/status or alert semantics for runner loading/load errors, anonymous
  account registration error, anonymous follow-up email error, and follow-up
  email saved status.
- Added focused web source tests for semantic progress, no range input for
  scale, scale option-id save path, scale-required alert wiring, and question
  control associations.
- Updated draft release notes and recorded manual screen reader/browser gaps in
  markdown/FOLLOW_UPS.md.
