Phase 56 - Glossary Question Search API Foundation

Prompt:
prompts/prompt_56.txt

Branch:
phase-56-question-search-api

Implementation summary:
- Added shared Admin Glossary question-search types in packages/shared/src/index.ts.
- Added GET /api/admin/glossary/question-search?q=... under the existing
  Admin-only Glossary router.
- Added query validation:
  - q is trimmed.
  - blank and one-character queries return 200 with no results.
  - limit defaults to 20.
  - limit is capped at 50.
  - invalid/non-positive limits return 400.
- Added apps/api/src/services/glossary.ts search helper using
  case-insensitive substring matching over survey_questions.question_text only.
- Search includes draft and published assessments, excludes retired
  assessments, and excludes soft-deleted surveys.
- Results include assessment/page/question context plus first-match start/end
  offsets.
- Results order by earliest match position, assessment title, page display
  order, question display order, and question id.
- Added focused API tests in apps/api/test/glossary.test.ts for auth,
  authorization, validation, status/deleted filtering, offsets, shape,
  metadata omission, limits, and ordering.
- Updated unreleased notes, PHASE_LOG, and FOLLOW_UPS.

Important scope notes:
- No Admin Glossary UI, tabs, debounce, loading/empty states, or rendering were
  implemented. That remains Phase 57.
- No result-to-entry workflow or prefilled Glossary creation wiring was

- No migrations, indexes, full-text search, fuzzy search, answer-option search,

API contract:
- GET /api/admin/glossary/question-search?q=<query>&limit=<optional>
- Admin-only via requireAuth + requireRole("admin") on the existing
  adminGlossaryRouter.
- Response shape:
  {
    query: string,
    minQueryLength: 2,
    limit: number,
    results: [
      {
        assessment: { id, title, status },
        page: { id, title, displayOrder } | null,
        question: { id, questionText, displayOrder },
        match: { start, end }
      }
    ]
  }

Review focus requested:

- Non-retired filtering: draft and published included, retired and

- Data exposure: no hidden tags, answer options, response data, or glossary

- Match offsets and deterministic ordering.
