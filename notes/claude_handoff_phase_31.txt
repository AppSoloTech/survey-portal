## Phase Scope

Implemented:

- Added migration `0030_anonymous_public_directory.sql`.
  - Adds `anonymous_survey_links.listed_in_public_directory boolean not null
    default false`.
  - Adds a partial index for listed anonymous links.
- Extended shared anonymous link contracts with `listedInPublicDirectory`.
- Added shared public directory response contracts:
  - `AnonymousSurveyDirectoryItem`
  - `AnonymousSurveyDirectoryResponse`
- Added Admin-only endpoint:
  - `PATCH /api/surveys/:id/anonymous-links/:linkId/public-directory`
  - Requires `requireAuth` and `requireRole("admin")`.
  - Accepts only boolean `listedInPublicDirectory`.
- Updated anonymous link behavior:
  - New links are unlisted by default via DB default.
  - Rotated replacement links are unlisted by default.
  - Disabled links have `listed_in_public_directory` cleared.
  - Rotated-out old links have `listed_in_public_directory` cleared.
- Added public endpoint:
  - `GET /api/anonymous-survey-directory`
  - Returns only links that are explicitly listed, enabled, unexpired, scoped to
    published non-deleted surveys, and have a decryptable public URL.
  - Payload includes only survey title, survey description, category name,
    expiration timestamp, listing timestamp, and public runner URL.
- Added public web route:
  - `/anonymous-surveys`
  - Outside `ProtectedRoute` and `AdminRoute`.
  - Lists eligible directory entries and links to existing
    `/anonymous-surveys/:token` runner URLs.
- Added Admin Setup control:
  - Per-link checkbox to opt an enabled anonymous link into or out of the
    public directory.
  - Displays a directory status pill when listed.
- Updated docs:
  - `markdown/DATA_MODEL_VISION.md`
  - `markdown/FOLLOW_UPS.md`
  - `markdown/PHASE_LOG.md`
- Expanded `apps/api/test/anonymousSurvey.test.ts`.

Non-goals respected:

- Did not list every enabled anonymous link by default.
- Did not add discovery for authenticated-only surveys.
- Did not implement anonymous assignment, recipient-list management, or email
  invitation workflows.
- Did not implement anonymous completion registration, sign-up conversion,
  email capture changes, or ownership conversion.
- Did not add public search/filtering beyond a simple list.
- Did not change anonymous attempt ownership, answer validation, reporting,
  hidden-tag behavior, CSV exports, or timing activity internals.
- Did not expose Phase 30 profile demographics/contact fields.

Known tradeoffs:

- Directory cards use the encrypted `public_token` reveal path already used for
  Admin repeat-copy. If a legacy enabled link has no decryptable public token,
  it is skipped by the directory even if manually flagged in the database.
- `listedAt` uses the anonymous link `updated_at`, so later link mutations can
  change the displayed listing timestamp. It is a display hint, not an audit
  trail.
- The public directory endpoint is unauthenticated by design and is not behind
  the anonymous token route limiter. Review whether that is acceptable for this
  simple listing endpoint or whether it should reuse/receive rate limiting.

Deferred work:

- Manual browser pass remains pending:
  - Admin creates a link and confirms it is unlisted by default.
  - Admin opts it into `/anonymous-surveys`.
  - Participant starts the listed survey.
  - Admin opts out, disables, or expires it and confirms it disappears.
  - Responsive pass at 375/768/1280px.
- No public search/filtering was added; keep as later scope if requested.
