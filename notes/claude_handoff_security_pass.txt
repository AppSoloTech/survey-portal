## Scope

Implemented:

- Added centralized security middleware:
  - `Content-Security-Policy`
  - production-only `Strict-Transport-Security`
  - `X-Frame-Options`
  - `X-Content-Type-Options`
  - `Referrer-Policy`
  - `Permissions-Policy`
  - `Cross-Origin-Opener-Policy`
- Added browser-origin validation for unsafe requests.
- Added CSRF token issuance through `GET /api/auth/csrf`.
- Added CSRF validation for browser unsafe requests that are not public
  anonymous survey endpoints.
- Kept `POST /api/anonymous-surveys/:token/*` endpoints CSRF-exempt because
  those routes use anonymous link bearer tokens plus per-attempt access tokens
  and do not rely on the auth cookie.
- Replaced in-memory auth/password-reset limit stores with
  `PostgresRateLimitStore`.
- Added email/account-aware rate-limit keys for login, registration, and public
  password-reset request attempts.
- Added `users.session_version` migration.
- Included `sessionVersion` in auth JWT payloads and rejected stale cookies in
  `requireAuth`.
- Bumped `session_version` after password reset and admin role changes.
- Expanded request URL redaction for token, secret, password, reset, attempt,
  and email-like query parameters.
- Split `/anonymous-surveys` and `/anonymous-surveys/:token` out of
  `AuthProvider` so public anonymous pages do not call `/api/auth/me` or render
  authenticated account/admin navigation.
- Added frontend CSRF token handling in the shared web API client.
- Removed the inline theme bootstrap from `apps/web/index.html`; moved initial
  theme selection into bundled module code.
- Added CI supply-chain checks:
  - `npm audit --omit=dev --audit-level=high`
  - `gitleaks/gitleaks-action`
  - Dependabot for npm and GitHub Actions.
- Added security tests for headers, CSRF pass/fail, origin rejection, password
  reset session invalidation, and role-change session invalidation behavior.
- Added `notes/security_pass.txt` with implementation notes and Azure checklist.

Non-goals:

- No full migration to Microsoft Entra authentication.
- No Azure portal/CLI mutations were performed.
- No Key Vault wiring was implemented in code; it remains an Azure deployment
  checklist item.
- No WAF, private endpoint, VNet, Defender, App Insights, or Azure RBAC changes
  were applied directly.
- No real email provider was added.
- No broad refactor of survey-builder/reporting routes was attempted.
- No public anonymous survey directory rate limiter was added in this pass.

Known tradeoffs:

- CSRF enforcement is browser-oriented: unsafe requests with `Origin` or
  `Referer` must pass origin and CSRF checks, while non-browser automation
  without those headers remains allowed.
- `style-src` still allows `'unsafe-inline'` to preserve current CSS/font
  behavior; `script-src` is restricted to `'self'`.
- Public anonymous endpoints remain CSRF-exempt because adding CSRF there would
  complicate unauthenticated token-link flows without protecting auth cookies.
- Session invalidation is coarse-grained per user via `session_version`, not a
  per-device session registry.
- The CI gitleaks action may need repository/organization approval or tuning on
  first GitHub run.
