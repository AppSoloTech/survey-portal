## Findings

### F1 — Medium Design — Conditional-target exclusion is global and unconditional

Finding:
Any question targeted by any `JUMP_TO_QUESTION` rule is excluded from ordinary linear flow. This is the intended interpretation for the manual test, but it is an authoring constraint that should be visible to admins.

Disposition:
Documented in the builder UI and phase notes.

Resolution:
Added a note beside the jump-rule builder: target questions are shown only when a jump rule reaches them; normal forward flow skips jump targets.

Deferred:
A more scoped per-branch reachability model is a larger branching-design change and is not part of this Phase 5 test-fix pass.

### F2 — Medium Maintainability — `resolveNextQuestion` duplicated across frontend/backend

Finding:
The navigation resolver existed in both the user dashboard and API route file, creating drift risk between the UI path and backend completion validation.

Disposition:
Fixed.

Resolution:
Hoisted `resolveNextQuestion` into `@survey-portal/shared` and updated both frontend and backend to import the shared implementation.

### F3 — Low — Custom tag suggestion duplicate becomes non-removable

Finding:
Custom suggestions that duplicate built-in or saved suggestions were merged under the first source and therefore lost the remove button.

Disposition:
Fixed.

Resolution:
Suggestion removability is now based on membership in `customTagPresets`, not the merged source label. Duplicated custom suggestions can be removed, revealing any underlying built-in or saved suggestion.

### F4 — Low Cosmetic — Stale preset wording after suggestion rename

Finding:
Buttons and labels now say suggestion, but toasts still said `Tag preset added` / `Tag preset removed`.

Disposition:
Fixed.

Resolution:
Updated notices to `Tag suggestion added` and `Tag suggestion removed`.

### F5 — Low Info — Saved suggestions aggregate tags across loaded surveys

Finding:
The tag suggestion library includes hidden tag vocabulary from all loaded admin surveys, not only the selected survey. The previous `Saved` label could imply current-survey-only scope.

Disposition:
Fixed.

Resolution:
Relabeled source text to `Saved in surveys`.
