3. ACCEPTABLE TRADEOFFS

(a) Manual-first plan with no new automation dependencies. Correct call. The
    prompt requires human approval for new dev dependencies, and the repo has
    only Vitest source/unit tests. The "Tooling Position" section is explicit
    that automated checks do not replace manual keyboard/AT testing and scopes
    the future Playwright+axe path behind approval. This is the honest and
    correct stance, not an under-delivery.

(b) Contrast documented as a workflow, not as measured pass/fail. Reasonable for
    a phase run without a real browser; the acceptance targets (WCAG 2.2 AA,
    not-by-color-alone, visible focus in both themes) are concrete enough to
    execute against later.

(c) Source-level testing convention unchanged. The plan correctly records that
    Vitest source checks do not render DOM or simulate AT, and positions the
    manual plan as the authoritative regression workflow until automation is
    approved.
