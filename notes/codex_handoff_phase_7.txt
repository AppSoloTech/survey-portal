Priority 1 (participant runner):

- `<=760px`: `.survey-workspace.survey-active .survey-list-panel` is hidden so

## Post-Review Fix Addendum

Codex review was recorded in `notes/codex_review_phase_7.txt` with one High
finding: hiding the survey list on mobile while a survey is active removed the
only in-progress exit path, because `Back to surveys` rendered only on the
completion panel. A participant on a question screen could not return to the
list without finishing the survey or reloading — and the Phase 7 prompt
explicitly required keeping a clear way back to the list.

Fix applied in `UserDashboard.tsx`:

- The active-question `survey-actions` row now renders a `Back to surveys`
  ghost button wired to the existing `onClose` callback, disabled while
  submitting — identical semantics to the completion panel's button. Closing
  mid-question discards only the current question's unsubmitted draft (the
  existing close behavior); saved answers persist server-side and resume
  works as before. The button is last in the action order and ghost-styled so
  it cannot be mistaken for the primary Next action, and the existing mobile
  full-width action rules apply to it automatically.

Re-validated after the fix: `npm run typecheck`, `npm run lint`,
`npm run build`, and `git diff --check` all passed.
