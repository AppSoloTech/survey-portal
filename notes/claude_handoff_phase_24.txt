## Phase Scope

Implemented:

- Added migration `0025_question_other_tags.sql`.
  - `question_other_tags` stores hidden tags directly against `survey_questions`.
  - It has an FK to `survey_questions`, unique `(question_id, tag_key, tag_value)`, length/nonblank checks, and a question index.
- Added shared/admin-only contracts:
  - `QuestionOtherTag`
  - `SurveyQuestion.otherTags`
  - `AdminAttemptAnswer.otherTags`
- Extended admin survey structure loading to include Other tags only when `includeHiddenTags` is true.
- Added draft-only admin builder routes:
  - `POST /api/surveys/:id/questions/:questionId/other-tags`
  - `PUT /api/surveys/:id/questions/:questionId/other-tags/:tagId`
  - `DELETE /api/surveys/:id/questions/:questionId/other-tags/:tagId`
- Create/update routes reject Other tags unless the question is `single_select` or `multi_select` and `allow_other = true`.
- Other tag writes register the key/value pair in `tag_definitions`.
- Survey duplication copies Other hidden tags to the cloned draft.
- Admin Questions UI renders add/edit/remove controls for Other hidden tags only on persisted choice questions with Allow Other enabled.
- Admin Results attempt detail displays Other hidden tags when `otherText` is present.
- CSV hidden-tag export includes Other hidden tags only on rows where Other was answered.
- Hidden-tag rollup counts Other tag events/respondents when `other_text` is non-null.
- Participant survey/start/answer payloads still omit Other hidden tags because public reads do not request hidden tags.
- Updated:
  - `markdown/DATA_MODEL_VISION.md`
  - `markdown/FOLLOW_UPS.md`
  - `markdown/PHASE_LOG.md`
- Added focused tests in `apps/api/test/choiceOtherAnswers.test.ts`.

Non-goals respected:

- No normal `answer_options` row for Other.
- No editable Other label/copy.
- No conditional logic triggered by Other.
- No hidden tags based on matching specific Other text values.
- No Other support for `scale`, `text`, or `integer`.
- No aggregation/normalization of identical Other text values.
- No changes to Phase 20 admin-user/profile/contact tooling or `/api/profile`.

Known tradeoffs:

- Other tags are resolved from current survey metadata at report time, matching existing answer-option/value-tag behavior. Adding/editing tags later can affect historical Other responses on the same survey.
- Disabling Allow Other hides the UI and create/update routes for Other tags, but existing rows are not deleted. If Allow Other is re-enabled on that draft question, previously configured tags return.
- Delete is allowed by tag id/question id for cleanup even if Allow Other is later disabled.
