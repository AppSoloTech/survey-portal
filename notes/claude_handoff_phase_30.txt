## Phase Scope

Implemented:

- Added `database/migrations/0029_user_profile_address.sql` with optional
  `user_profiles.address` and a 240-character check constraint.
- Updated shared profile response types:
  - `UserProfile` now exposes `contactNumber`, `address`, and timestamps.
  - `UpdateCurrentUserProfileResponse` now returns the updated `AuthUser` plus
    saved profile.
- Updated `PUT /api/profile` to validate and persist:
  - first name
  - last name
  - phone number via existing `contact_number` / `contactNumber`
  - address
- Kept optional profile-field blank normalization to `null`.
- Kept profile update partial-merge semantics.
- Kept profile routes self-service-only; no `/api/profile/:id` behavior added.
- Updated Settings to edit first name, last name, address, and "Phone number".
- Updated the auth context so Settings can refresh the current client auth user
  immediately after save; account-menu identity updates without reload.
- Preserved the existing Settings survey stats and password-reset panel/cooldown.
- Updated admin user detail to display address and phone read-only when present.
- Removed legacy `preferredContactMethod` and `contactNotes` from visible shared
  response types, Settings, and Admin user detail. Existing database columns/data
  are preserved.
- Updated `markdown/DATA_MODEL_VISION.md`, `markdown/FOLLOW_UPS.md`, and
  `markdown/PHASE_LOG.md`.

Non-goals respected:

- No structured address fields.
- No profile photos or uploads.
- No admin demographic editing.
- No CRM/account-management workflow.
- No additional demographic fields beyond full name, address, and phone number.
- No session invalidation after name/profile edits.
- No password reset behavior changes.

Known tradeoffs:

- `address` allows up to 240 characters while the existing phone/name profile
  validation remains 120 characters.
- Existing legacy columns (`organization`, `job_title`, `location`,
  `preferred_contact_method`, `contact_notes`) remain in the database and are
  tracked for a later cleanup decision.
- Manual browser checks were not run in this environment; they are logged as a
  follow-up.
