## Phase Scope

Implemented:

- Added shared response types:
  - `AdminUserDetailResponse`
  - `AdminUserPasswordResetResponse`
- Exported Phase 19 profile/stat read helpers:
  - `fetchUserProfile`
  - `fetchRegisteredUserSurveyStats`
- Added admin-only routes:
  - `GET /api/admin/users/:id`
  - `POST /api/admin/users/:id/password-reset`
- Admin detail returns:
  - safe user account projection: id, first name, last name, email, role, createdAt
  - read-only profile fields: contactNumber, preferredContactMethod, contactNotes, profile timestamps
  - registered-user survey stats using Phase 19 semantics
- Admin reset uses the Phase 18 password reset service and returns only the generic reset message.
- Added `/admin/users/:userId` UI linked from the existing admin user table.
- Added reset initiation feedback in the admin detail UI.
- Added API tests for admin-only access, unknown users, secret exclusion, no-profile nulls, anonymous-attempt isolation, reset token row creation, and admin self-reset.
- Updated `markdown/FOLLOW_UPS.md` and `markdown/PHASE_LOG.md`.

Non-goals respected:

- No reset tokens, reset URLs, token lookup keys, token hashes, or password hashes are returned to admins.
- No account impersonation was added.
- No admin demographic editing was added.
- No bulk user actions were added.
- No CRM-style workflow was added.
- `/api/profile` was not extended with user-id params or admin behavior.
- Phase 19 completion-rate semantics were not changed.

Known tradeoffs:

- Admin reset returns the same generic password-reset message used by the public/self-service flows even though the admin route already knows whether the target user exists. This keeps the response secret-free and simple.
- Admin self-reset is allowed, matching the Phase 20 open-question default, but still exposes no token material.
- Manual browser checks were not run in this environment; they are logged as follow-up.
- In disabled/no-op email mode, the admin route creates the reset token row through the service, but the admin response still does not include a dev reset link.

Deferred work:

- Manual browser pass over admin user list/detail/reset at 375/768/1280px.
- Claude review output file creation.
