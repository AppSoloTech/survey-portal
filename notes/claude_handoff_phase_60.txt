# Claude Handoff — Phase 60 Issue Profile Emoji Burst

## Summary

Phase 60 adds admin-managed emoji metadata to tag catalog definitions and uses
that metadata to create a participant-safe Ready to submit emoji burst from the
issue profile thermometer.

The privacy boundary remains the main review concern: participants and
anonymous runners receive only emoji/count aggregate items and a total count.
They must not receive tag keys, tag values, tag definition IDs, tag group names,
category labels, selected tag names, or hidden tag arrays.

## Main Changes

- Added migration `database/migrations/0039_tag_definition_emoji.sql`.
- Added nullable `emoji` to shared `TagDefinition` and hidden-only tag record

- Updated the Admin Tags catalog create/edit UI and API to manage optional emoji

## Semantics

- Emoji counts are frequency-based.
- A matched selected-option tag contributes one count for its configured emoji.
- A matched Other tag contributes when `otherText` is nonblank.
- A matched text/integer value tag contributes when `valueTagMatchesResponse`
  is true.
- Tags without emoji are ignored.
- Multiple tag definitions with the same emoji aggregate into one emoji item.
- Off-path/pruned responses do not contribute emoji counts.
- Burst particles are capped at 40 and weighted by emoji frequency.
- The live region announces only the Ready to submit collected-details event;
  per-answer fill changes remain silent unless the progressbar is queried.
- Static chips show up to eight emoji types plus a `+N` overflow indicator.
