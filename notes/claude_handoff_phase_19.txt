## Phase Scope

Implemented:

- Added `database/migrations/0022_user_profiles.sql` with a relational `user_profiles` table keyed to `users.id`.
- Added shared profile/stat response types:
  - `UserProfile`
  - `CurrentUserSurveyStats`
  - `CurrentUserProfileResponse`
  - `UpdateCurrentUserProfileResponse`
- Added current-user-only profile APIs:
  - `GET /api/profile`
  - `PUT /api/profile`
- Validates optional profile fields as text with a 120-character maximum, trims whitespace, and stores blank values as `null`.
- Derives registered current-user survey stats:
  - available published surveys with no active/completed registered attempt for the current user
  - in-progress registered attempts
  - completed registered attempts
  - last registered activity timestamp
  - rounded completion rate
- Keeps anonymous attempts out of stats by filtering stats from `survey_attempts.user_id = req.user.id`.
- Extends `/settings` with account details, optional profile fields, current-user survey stats, and the existing password reset panel/cooldown.
- Added focused API tests for auth rejection, profile persistence, current-user-only isolation, validation bounds, stats counts/rate, and anonymous-attempt exclusion.
- Updated `markdown/DATA_MODEL_VISION.md`, `markdown/FOLLOW_UPS.md`, and `markdown/PHASE_LOG.md`.

Non-goals respected:

- No admin profile viewing/editing tools were added.
- No admin password reset initiation was added.
- No profile photos or uploads were added.
- No public profile pages were added.
- No speculative demographics beyond organization/department, job title/role, and location/region were added.

Known tradeoffs:

- `PUT /api/profile` returns only the saved profile object; callers can re-fetch `GET /api/profile` when stats or auth-user data are needed.
- Completion rate is rounded whole-percent over available + in-progress + completed visible survey units.
- Manual browser checks were not run in this environment; they are logged as a follow-up.
