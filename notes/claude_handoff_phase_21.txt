## Phase Scope

Implemented:

- Added migration `0024_choice_other_answers.sql`.
  - `survey_questions.allow_other boolean not null default false`
  - check constraint limiting `allow_other` to `single_select` and `multi_select`
  - `survey_response_answers.other_text text`
  - check constraint bounding nonblank Other text to 240 characters
- Added shared contracts:
  - `SurveyQuestion.allowOther`
  - `SurveyResponseAnswer.otherText`
  - `SurveyAnswerRequestPayload`
  - `SurveyPageAnswerRequestPayload`
  - `AdminAttemptAnswer.otherText`
  - `SurveyReportQuestionStat.otherResponseCount`
- Added builder create/update validation for `allowOther`.
  - Allowed only on `single_select` and `multi_select`
  - Defaults false for existing and newly created unsupported questions
  - Preserved through survey duplication
- Added admin UI Allow Other toggle on draft single-select and multi-select questions only.
- Added participant UI system-generated Other row and text input when enabled.
- Added answer validation:
  - single-select required accepts exactly one standard option or one nonblank Other response
  - single-select rejects standard + Other
  - multi-select required accepts at least one standard option or nonblank Other response
  - multi-select accepts standard + Other together
  - selected Other with blank text is rejected
  - Other answers are rejected on unsupported question types or questions without `allowOther`
- Stored Other text in `survey_response_answers.other_text`, not in `answer_options`.
- Reporting changes:
  - response content checks include `other_text`
  - per-question stats include `otherResponseCount`
  - attempt detail returns `otherText`
  - CSV adds an `other_text` column separate from `selected_options`
  - hidden tags are still derived only from selected real answer options and value tags
- Updated docs:
  - `markdown/DATA_MODEL_VISION.md`
  - `markdown/FOLLOW_UPS.md`
  - `markdown/PHASE_LOG.md`

Non-goals respected:

- No normal `answer_options` row is created for Other.
- No hidden tags on Other.
- No conditional logic triggered by Other.
- No Other support for `scale`, `text`, or `integer`.
- No changes to Phase 20 admin user tools, password reset, profile contact fields, or `/api/profile`.
- No cleanup of old Phase 19 profile columns.
