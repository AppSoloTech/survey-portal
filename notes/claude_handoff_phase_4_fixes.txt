## Fix Scope

Backend changes to review:

- `apps/api/src/routes/surveys.ts`
  - `DELETE /api/surveys/:id/questions/:questionId`
    - now fetches the survey first
    - returns 404 when the survey does not exist
    - returns 409 when the survey status is not `draft`
    - only deletes questions for draft surveys
  - `DELETE /api/surveys/:id/questions/:questionId/options/:optionId`
    - now fetches the survey first
    - returns 404 when the survey does not exist
    - returns 409 when the survey status is not `draft`
    - only deletes answer options for draft surveys
  - `shiftQuestionDisplayOrdersForInsert`
    - now uses two statements:
      - park existing rows with a positive offset
      - set final display orders
    - no longer updates the same rows through a data-modifying CTE plus outer update in one statement
  - `shiftOptionDisplayOrdersForInsert`
    - same two-statement park-then-set change as question shifting
  - `validateConditionalRuleReferences`
    - now rejects target questions whose `display_order` is less than or equal to the source question
  - `validateSurveyCanPublish`
    - now treats rules as invalid when the target question is not after the source question

Frontend changes to review:

- `apps/web/src/pages/AdminDashboard.tsx`
  - disables question delete buttons when the survey is not draft
  - disables answer option delete buttons when the survey is not draft
  - add-rule target dropdown only lists questions after the selected source question
  - add-rule button is disabled when no forward target exists
  - existing rule editor target dropdown only lists questions after the selected source question
  - tag dropdown fixes from manual UI testing:
    - add-tag fields start empty
    - saved tag rows keep their persisted key/value after unrelated saves
    - add-rule fields start empty

Documentation changes to review:

- `markdown/FOLLOW_UPS.md`
  - C1 moved to completed
  - S1 code hardening moved to completed
  - S2 moved to completed
  - live insert-at-position/reorder smoke test remains active
- `markdown/PHASE_LOG.md`
  - Phase 4 status updated after Claude review fixes
