## Phase Scope

Implemented:

- npm workspace foundation with `apps/web`, `apps/api`, and `packages/shared`.
- React + Vite frontend shell.
- Placeholder routes:
  - Home
  - Login
  - Register
  - User Dashboard
  - Admin Dashboard
  - Not Found
- Express + TypeScript backend.
- `/api/health` route.
- PostgreSQL connection module using `pg`.
- Environment loading from root `.env` and workspace `.env`.
- `RUN_ENV=dev|prod` database selection.
- Support for local split PostgreSQL variables and database URL variables.
- Basic request logging.
- Basic error handling middleware.
- Development CORS constrained to `WEB_ORIGIN`.
- Express static serving for the built React app.
- Minimal shared types.
- Minimal database scaffolding with an `app_health_check` migration.
- Root scripts for dev, build, typecheck, lint, and start.
- `.env.example`, `.nvmrc`, root TypeScript config, ESLint config, and README updates.
- Workflow documentation updates requiring Claude handoff and review artifacts after each phase implementation.

Non-goals:

- Real registration.
- Real login.
- JWT issuing.
- bcrypt hashing.
- Protected routes.
- Survey builder.
- Survey taking.
- Admin reporting.
- Conditional logic implementation.
- CSV export.
- Azure deployment.
- Full survey schema.

Known tradeoffs:

- API `dev` script builds TypeScript then runs `node dist/server.js` instead of using a watch/dev runtime. This avoids `tsx` IPC failures in the current environment.
- The health endpoint reports database availability but still returns HTTP 200 if the database is unavailable. This keeps the endpoint usable during local scaffolding, but Claude should flag if this should change before deployment.
- Vite is used for the frontend build tool. This is conventional and lightweight, but it is an added tool.
- The initial migration is intentionally tiny and does not start the real survey schema.

Deferred work:

- Browser-based visual inspection.
- Claude review fixes after this review.
- Real auth and authorization.
- Survey data model implementation.
- Migration runner choice beyond ordered SQL files.
- Production Azure deployment configuration.
