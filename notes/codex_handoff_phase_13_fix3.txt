The finding (P3)
----------------
fix #2 keyed the "reveal same-page question targets" exception on
normalFlowExcludedPageIds.has(fromQuestion.pageId) — i.e. only JUMP_TO_PAGE
branch pages. But the runtime sets wasReachedByJump for ANY firable navigation
target (resolveTriggeredNavigationPage accepts JUMP_TO_PAGE and JUMP_TO_QUESTION;
the resolved target page is added to jumpedToPageIds). So a page reached by a
skip-flow JUMP_TO_QUESTION, containing another same-page question-level jump
target, still diverged (graph skipped the same-page target; runtime revealed it).

The fix (apps/web/src/components/admin/surveyFlowGraph.ts)
---------------------------------------------------------
Re-keyed the same-page exception on whether the FROM-question is conditional-only
(skipTargetQuestionIds), instead of on its page being a JUMP_TO_PAGE branch:

    const fromPageReachedByJump = conditionalOnlyQuestionIds.has(fromQuestion.id);
    ...within same page: if (!fromPageReachedByJump && excludedQuestion) continue;

Rationale: skipTargetQuestionIds is exactly "questions reachable only via a jump"
(JUMP_TO_QUESTION skip targets ∪ all questions on JUMP_TO_PAGE branch pages). A
conditional-only question's page is, by definition, only entered by a jump — the
runtime's wasReachedByJump — so the rest of that page is revealed regardless of
question-level skips. This now covers JUMP_TO_QUESTION-reached pages and
JUMP_TO_PAGE branch pages uniformly. It is also self-consistent with the node's
own label: conditional-only nodes render "Skipped in normal flow. After a jump
lands here, the survey continues to X" (post-jump context), while normally-
reached nodes render "Normal flow continues to X" (normal-entry context, which
correctly skips same-page question targets).

skipTargetQuestionIds is passed into resolveGraphNormalNext as a 5th arg
(conditionalOnlyQuestionIds). node.normalNextQuestionId and buildAdjacency still
share the one precomputed successor map, so reachability tracks the fix (Q3 is
reachable via Q2).

Known modeling boundary (not a regression)
------------------------------------------
A single static successor per node cannot represent a question that is reachable
BOTH by normal flow AND by a jump-landing-then-walk (e.g. a normal question that
shares a page with a jump target). For such NON-conditional-only questions the
map shows the normal-ENTRY successor (skips the same-page jump target), matching
its "Normal flow continues to..." label; the post-jump sub-path successor is not
separately rendered. This is inherent to the one-successor-per-node graph model,
not a wasReachedByJump miss — every question that is only reachable by jump is
conditional-only and so gets the post-jump successor.
