Phase 44 - Form Feedback And Required Field Accessibility

Implementation summary:

- Added `apps/web/src/components/AlertMessage.tsx`, a small status primitive
  that renders blocking errors as `role="alert"`/assertive live regions and
  success/info messages as `role="status"`/polite live regions while preserving
  the existing `.status` visual classes.
- Added `apps/web/src/components/FormField.tsx`, a small field wrapper that
  provides stable ids, visible required/optional markers, helper text, error
  text, `aria-describedby`, and `aria-invalid`.
- Applied the form pattern to public/registered-user forms:
  - Login
  - Register
  - Forgot password
  - Reset password
  - Account settings profile form
- Kept backend/API validation behavior unchanged. Existing server/form errors
  remain one visible alert, while relevant controls now reference the alert and
  can be marked invalid.
- Added field-specific phone validation feedback in Account settings, tied to
  the phone input through the shared field wrapper.
- Added discoverable disabled-state explanations for:
  - invalid/expired reset-password links
  - account password-reset cooldown
- Converted scoped public/user statuses to `AlertMessage` where practical:
  - home API health check
  - protected user-route session loading
  - dashboard loading/empty/error/search-empty states
  - category survey loading/error states
  - anonymous directory loading/error states
  - account loading/success/error states
  - auth success/error states
- Updated shared `PaginationRow` so the visible page status is a polite,
  atomic `role="status"` live region.
- Updated `ToastProvider` so toasts are not generic clickable buttons:
  success toasts render inside a persistent polite status live region, error
  toasts render inside a persistent assertive alert live region, each toast has
  a dedicated visible dismiss button, and auto-dismiss timing is longer.
- Restored staggered reveal behavior for migrated auth fields through a
  `FormField` `reveal` prop.
- Added focused source-level tests in
  `apps/web/src/components/FormFeedback.test.ts`.
- Updated draft release notes and Phase 44 project memory/follow-ups.
