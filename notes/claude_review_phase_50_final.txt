2. SUGGESTED IMPROVEMENTS
============================================================
(a) classifyBottleneck still has a hardcoded `maxActive >= 20` DB-pressure threshold
    (reporting.mjs:57) that is not scaled by expectedPoolCeiling like the app_pool branch
    is. On a multi-instance app or a larger pool this constant is arbitrary. Low priority;
    consider expressing it relative to the ceiling or making it configurable.

(b) Active-connection visibility on Azure (residual from prior 2b). The fix correctly leads
    with maxTotalConnections and surfaces a visibility caveat in the recommendation, which
    is the right mitigation. The residual remains that maxActiveConnections (the `state`
    filter) silently undercounts other-role backends unless the load-test role holds
    pg_monitor / pg_read_all_stats. Recommend the README state plainly: "grant pg_monitor to
    the load-test role for accurate active-connection classification, or rely on total
    connections." (If the README already says this, treat as satisfied.)

(c) SIGINT abort is best-effort and can race the normal completion path / an in-flight
    pool query (run.mjs handler vs finishAndPersist vs k6 child receiving the same signal
    via the process group). It is guarded by a `handled` flag and try/catch, so the worst
    case is a duplicate or skipped abort write, not corruption. Acceptable as-is; noting for
    completeness.

(d) The default ramp peaks at 50 VUs. That is a sensible, safe default, but it may be below
    the knee for a production SKU, so operators hunting the true upgrade threshold will need
    to raise targets. Worth one line in the README pointing at --ramping-stages for pushing
    past 50.

  reason. Consistent with the no-new-Azure decision; SQL stats are the in-scope substitute.
- Admin password is stored in the gitignored manifest and passed to k6 via env. Acceptable

  distinguishing broad DB pressure from app-pool pressure.
- Ramping is opt-in-by-profile (non-smoke) with a conservative default curve; smoke stays
  constant-vus at 1.
