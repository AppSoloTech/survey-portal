##### Resolution Notes

Addressed in the Phase 5 test-fix pass:

1) Hidden tag fields now use dropdowns with `Custom key...` and `Custom value...` escape hatches. Existing saved tags render their saved key/value directly, and new tags are saved only through `Add hidden tag`.
2) Answer option editing was refactored so option text has its own `Save option text` action, while hidden tags have separate `Add hidden tag`, `Save tag`, and `Remove tag` actions.
3) Added `Back to top` in the bottom survey status area.
4) Integer questions now show a numeric placeholder and helper text explaining that only whole numbers are accepted.
5) The Hidden Tags section is now a tag suggestion library with source labels for built-in, saved, and custom suggestions.
6) Custom tag suggestions are the only removable suggestions; built-in and saved suggestions are labeled but not removable.
7) Conditional jump target questions are now treated as conditional-only in survey navigation. They are skipped by ordinary linear flow unless reached by a matching jump rule. This logic was updated in both the frontend runner and backend completion/current-question helpers.
8) Added a jump-rule target-flow toggle. Admins can leave `Skip target in normal flow` checked for conditional-only target questions, or uncheck it to keep the target question in the normal question flow.
