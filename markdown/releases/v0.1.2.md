# v0.1.2 - Participant Runner

Release date: 2026-06-11

Summary: Delivers the first authenticated survey-taking experience, including answer persistence, navigation, validation, and completion handling.

## Added

- Authenticated users can start or resume published surveys from the dashboard and move through a focused one-question-at-a-time runner.
- Answer saving supports text, integer, single-select, multi-select, required-question validation, active-attempt reuse, completion locking, and submit flow.
- Participant progress, previous/next navigation, and answer restoration make interrupted survey attempts recoverable for registered users.
- Attempt and response services centralize runtime behavior so reporting and later anonymous flows can reuse the same validation model.

## Changed

- Concurrent starts now return an existing active attempt, and abandoned or completed attempts cannot accept additional answers.

## Operational Notes

- Roadmap source: Phase 3 in `markdown/PHASE_LOG.md` and `prompts/prompt_3.txt`.
- This milestone is registered-user only; anonymous access and public links arrive in later roadmap entries.
