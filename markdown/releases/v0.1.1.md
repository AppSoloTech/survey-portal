# v0.1.1 - Accounts And Survey Model

Release date: 2026-06-09

Summary: Adds the user account system and the normalized survey schema/API foundation needed for secure participant and Admin workflows.

## Added

- User registration, login, logout, session persistence, protected routes, Admin route checks, and role-aware dashboard entry points.
- Database tables for users, surveys, questions, answer options, hidden answer tags, conditional rules, attempts, responses, and selected options.
- Foundational survey APIs for listing, reading, creating, and updating surveys with Admin-only visibility into draft data and hidden tags.
- Local seed data with an Admin user, a published test survey, MVP question types, hidden tags, and a sample jump rule.

## Security

- Passwords are stored with bcrypt, JWT auth is cookie-backed, Admin-only endpoints require server-side role checks, and standard users receive participant-safe survey payloads.

## Operational Notes

- Roadmap source: Phases 1-2 in `markdown/PHASE_LOG.md` and `prompts/prompt_1.txt` through `prompts/prompt_2.txt`.
- The survey schema remains intentionally additive so later migrations can extend the MVP without rewriting existing data.
