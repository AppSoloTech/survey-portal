# v0.2.0 - Glossary Definitions

Release date: 2026-06-27

Summary: Adds glossary definition tools for Admins and inline glossary definitions for survey participants.

## Added

- Admin glossary editors can request Merriam-Webster definition suggestions, apply a suggestion, edit it before saving, or keep a manual definition.
- Dictionary lookups run server-side behind environment configuration and keep provider keys out of browser code.
- Survey participants can hover, focus, or tap Admin-approved glossary terms in question prompts to read definitions inline.
- Registered and anonymous survey runner payloads now include enabled participant-safe glossary entries without Admin-only source metadata.
- Admin survey preview renders glossary terms consistently with the participant runner.

## Operational Notes

- Dictionary suggestions are disabled by default with `DICTIONARY_PROVIDER=disabled`.
- To enable Merriam-Webster suggestions, configure `DICTIONARY_PROVIDER=merriam-webster` and `MERRIAM_WEBSTER_COLLEGIATE_API_KEY` in the environment.
- Glossary rendering is informational only; survey answers, navigation, reporting, CSV exports, and hidden tags are unchanged.
