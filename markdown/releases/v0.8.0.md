# v0.8.0 - User-Facing Accessibility

Release date: 2026-06-27

Summary: Improves participant survey-taking, public routes, account forms, route context, dialogs, statuses, and repeatable accessibility verification for user-facing workflows.

## Added

- Route accessibility manager updates browser titles, announces navigation, and moves focus to main content after public and registered-user route changes.
- Skip-to-main link, shared accessible modal primitive, AlertMessage, FormField, improved toast semantics, and live-region pagination/status patterns.
- Repeatable public and registered-user accessibility test plan plus durable accessibility primitive usage guidance for future phases.

## Fixed

- Survey scale questions now use native radio-style choices so unanswered required scales are not exposed as the minimum value.
- Runner prompts, help text, validation errors, save/submit states, anonymous follow-up errors, and progress context are more reliably associated and announced.
- Auth, reset, account, dashboard, category, public directory, glossary, and repeated action controls now expose clearer accessible names and state.

## Operational Notes

- Roadmap source: Phases 42-46 in `markdown/PHASE_LOG.md`, `prompts/prompt_42.txt` through `prompts/prompt_46.txt`, and `prompts/accessibility_remediation_prompt_plan.txt`.
- This milestone focuses on public and registered-user accessibility; Admin-only remediation remains separately trackable.
