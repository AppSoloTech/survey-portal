# v0.7.0 - Glossary Definitions

Release date: 2026-06-27

Summary: Completes the glossary arc with Admin dictionary suggestions and accessible inline definitions for survey participants.

## Added

- Admin glossary editors can request Merriam-Webster definition suggestions, apply a suggestion, edit before saving, or continue with manual definition entry.
- Dictionary lookups run server-side behind environment configuration and keep provider credentials out of browser code.
- Registered and anonymous survey runner payloads include enabled participant-safe glossary entries without Admin-only source metadata.
- Survey participants can hover, focus, or tap Admin-approved glossary terms in question prompts to read definitions inline.
- Admin survey preview renders glossary terms consistently with the participant runner.

## Security

- Dictionary suggestions are Admin-only, mocked in tests, disabled by default, and require environment-held credentials before live provider access is enabled.

## Operational Notes

- Roadmap source: Phases 40-41 in `markdown/PHASE_LOG.md` and `prompts/prompt_40.txt` through `prompts/prompt_41.txt`.
- Glossary rendering is informational only; survey answers, validation, navigation, reporting, CSV exports, hidden tags, and response storage are unchanged.
