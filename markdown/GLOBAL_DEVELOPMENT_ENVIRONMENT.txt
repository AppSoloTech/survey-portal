# Global Development Environment

This file records environment assumptions for the Survey Portal MVP.

It should be updated when tooling, deployment targets, or local setup requirements change.

---

## Project Type

Full-stack TypeScript web application:

- React frontend
- Node.js + Express backend
- PostgreSQL database
- Express serves the production React build
- REST APIs live under `/api`

---

## Runtime And Package Manager

Use:

```txt
Node.js 24.15.0
npm 11.12.1
```

The Node.js version is recorded in `.nvmrc` and `package.json`.

Use npm unless the repository later commits a different lockfile and updates this document.

Expected root commands after scaffolding:

```bash
npm install
npm run dev
npm run typecheck
npm run build
npm run lint
```

If a command is not yet available, document that in the phase log instead of inventing a replacement.

---

## Local Environment Variables

Local secrets belong in `.env`.

The repository should commit `.env.example` with placeholder values only.

Expected variables may include:

```txt
NODE_ENV=
RUN_ENV=
PORT=
API_HOST=
WEB_ORIGIN=
DATABASE_URL=
LOCAL_DATABASE_URL=
HOSTED_DATABASE_URL=
DATABASE_SSL_CA_PATH=
DATABASE_SSL_CA=
JWT_SECRET=
JWT_EXPIRES_IN=
TRUST_PROXY_HOPS=
AUTH_RATE_LIMIT_WINDOW_MS=
AUTH_LOGIN_RATE_LIMIT_MAX=
AUTH_REGISTER_RATE_LIMIT_MAX=
ADMIN_FIRST_NAME=
ADMIN_LAST_NAME=
ADMIN_EMAIL=
ADMIN_PASSWORD=
```

If the implementation uses split PostgreSQL variables instead of `DATABASE_URL`, document the exact variables in `.env.example` and keep the database connection module consistent.

Never commit real credentials, production secrets, or client data.

---

## Database Assumptions

Local development uses PostgreSQL.

Production target:

- Azure PostgreSQL Flexible Server

Database work should use explicit migrations. Seed data must be fake, minimal, and safe to commit.

---

## Hosting Assumptions

MVP production target:

- Azure App Service
- Azure PostgreSQL Flexible Server

Azure deployment is not required until a phase explicitly asks for it.

Production configuration belongs in Azure App Service environment variables, not in source control.

---

## Repository Hygiene

Keep generated files out of source control unless they are intentionally part of the app.

Expected ignored files include:

- `.env`
- `.env.*`
- `node_modules/`
- build output
- coverage output
- local logs

`.env.example` should remain tracked.

---

## Validation Baseline

Once the project is scaffolded, every implementation phase should run the relevant available checks:

- install dependencies when needed
- typecheck
- build
- lint
- backend startup
- frontend startup
- API health check
- relevant manual workflow checks

Record unavailable or failing checks in `markdown/PHASE_LOG.md`.

---

## Open Environment Decisions

Resolved during Phase 0:

- Node.js version: 24.15.0
- npm version: 11.12.1
- frontend build tool: Vite
- database migration format: ordered SQL files under `database/migrations`
- backend query approach: `pg` connection pool with parameterized SQL
- local API port: 3000
- local web port: 5173
- local API host: 127.0.0.1
- local development process: root `npm run dev` starts both API and web workspaces
