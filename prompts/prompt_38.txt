# Scope

Implement:

* Keep each question editor as the parent container labeled by
  `formatQuestionLocator`, such as `P1-Q1`.
* Add accessible collapsible subsections inside `QuestionEditor`.
* Default state:
  * `Question details` open.
  * `Save as question template` collapsed.
  * `Hidden tags for answers` collapsed for text/integer questions.
  * `Answer options` or `Scale values` collapsed for option-backed questions.
  * `Hidden tags for Other` collapsed when applicable.
* Section labels should include useful counts where natural, for example
  `Answer options (3)`, `Hidden tags for answers (2)`, `Other tags (1)`.
* Preserve all existing edit, lock, published-survey, template-saving, and
  hidden-tag behavior.
* Keep section state local to the rendered editor; do not persist section state
  across sessions.
* Ensure keyboard and screen-reader accessibility with native `details` behavior
  or equivalent `aria-expanded` behavior.
* Keep styling consistent with the existing admin builder.
* Update `markdown/releases/unreleased.md`, `markdown/PHASE_LOG.md`, and
  `notes/claude_handoff_phase_38.txt`.

## Non-Goals

Do not implement:

* API contract, database schema, survey structure, reporting, template, or
  conditional logic changes.
* Moving question editing to another route.
* Removing existing question, option, template, or tag capability.
* Per-admin saved preferences for expanded/collapsed sections.
* A broader visual redesign.

## Assumptions

* Item 4 is intentionally separate because it touches dense editor structure and
  needs its own manual QA pass.

# Acceptance Criteria

The phase is ready for review when:

* The question page no longer shows all heavy tooling expanded by default.
* Admins can expand/collapse each subsection independently inside each question.
* The `P1-Q1` parent header remains visible and meaningful.
* Published surveys still lock structural controls while allowing the
  already-approved hidden-tag maintenance behavior.
* Text, integer, single-select, multi-select, and scale questions all render
  appropriate sections.
* Empty states and action buttons remain reachable when their section is
  expanded.
