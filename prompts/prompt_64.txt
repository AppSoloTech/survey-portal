# Scope

Implement:

* Add a detachable thermometer top/cap element to the Ready to submit burst.
* Use a larger SVG thermometer stage for the Ready to submit state so the tube,
  bulb, fill, crack, glow, burst rays, and explosion origin can be positioned
  precisely. Prefer a classic thermometer silhouette with a continuous
  blue-to-red warming fill, intact in-progress cap, broken Ready to submit rim,
  and subtle liquid polish such as a meniscus.
* Add 3-5 small cap/glass shards that fly outward from the thermometer top.
* Add richer spark and puff particles around the same burst origin.
* Keep the existing emoji particles bursting from the thermometer top.
* Use the existing `gsap` dependency exported from
  `apps/web/src/motion/motion.ts` to run one coordinated timeline.
* Preserve the existing burst replay behavior when returning to Ready to submit.
* Preserve reduced-motion behavior: skip flying/breaking particles while
  keeping the static collected emoji chips visible.
* Preserve screen-reader behavior: keep the meaningful polite announcement and
  do not announce decorative particles.
* Update release notes, phase log, follow-ups, and handoff.

## Non-Goals

Do not implement:

* A new npm package unless GSAP proves unusable.
* Canvas, WebGL, or Three.js explosion effects.
* Participant-visible hidden tag metadata, category names, counts, IDs, or
  selected tag names.
* Score, severity, grade, violation-count, or recommendation semantics.
* Backend, API, shared type, database, hidden-tag, admin-tag, reporting, CSV, or
  release-version changes.

## Assumptions

* `gsap` is the preferred animation library because it is already installed and
  used by the web app.
* The thermometer should look like it breaks open visually, but the effect is
  decorative only.
* Accessibility semantics and participant-safe payloads do not change.
* The static emoji collection remains the reduced-motion and no-animation
  fallback.

## Open Questions

None.

# Acceptance Criteria

The phase is ready for review when:

* The Ready to submit animation visibly detaches the thermometer top.
* 3-5 cap/glass shards and richer spark/puff graphics appear from the
  thermometer top without layout shift.
* Existing emoji particles still burst from the thermometer top-center and
  remain capped.
* The burst can replay through the existing burst-key behavior.
* Reduced-motion users do not receive flying/breaking particles and still see
  the static collected emoji chips.
* The issue-profile progressbar, value text, and polite live-region behavior
  remain intact.
* Participant-facing copy remains abstract and non-scoring.
* Hidden tag identities and admin metadata remain absent from participant and
  anonymous UI/payloads.
* Every received feedback item is fixed, deferred with a reason, or blocked by
  a documented question.
