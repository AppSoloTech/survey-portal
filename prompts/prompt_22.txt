# Received Feedback

Add optional question definitions/help text. Administrators should be able to
enter explanatory text for each question. If provided, the survey-taking
interface should display an information icon adjacent to the question text.
Selecting the icon should display the definition/help text in a tooltip or
popover without navigating away. The definition is informational only and should
not affect survey logic or reporting.

---

# Intake Classification

* UX feature
* participant-facing survey behavior
* likely frontend-only if existing `helpText` persistence is sufficient

Expected impact:

* admin question editor copy
* participant runner display
* accessible tooltip/popover component
* possibly preview display

No database change is expected because `survey_questions.help_text` already
exists.

---

# Scope

Implement:

* Confirm existing `helpText` persists through create/update/read APIs.
* Rename or clarify admin UI copy to "Definition/help text" where useful.
* Participant info icon adjacent to the question prompt when help text exists.
* Accessible tooltip/popover that works with click/tap and keyboard focus.
* Admin preview behavior consistent with the participant runner where practical.

## Non-Goals

Do not implement:

* a new schema field if `helpText` is sufficient
* changes to validation, conditional logic, reporting, or exports
* participant navigation away from the survey
* help text as answer data
* rich text/HTML definitions

## Assumptions

* `helpText` is the durable storage field for definitions.
* Definition text is plain text.
* Tooltip/popover content should be visible only when text exists.

## Open Questions

* None unless inspection shows `helpText` cannot support this behavior.

---

# Acceptance Criteria

The phase is ready for review when:

* Admins can create and edit definition/help text.
* Participant runner shows an info icon only for questions with help text.
* Tooltip/popover displays help text without changing survey state.
* Keyboard and mobile interactions work.
* Survey logic, reporting, and exports are unchanged.
