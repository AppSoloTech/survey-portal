# Phase 22 — Superseded: Question Definitions Tooltip Foundation

## Status

Do not implement this prompt as originally written.

The original Phase 22 idea was a small per-question `helpText` tooltip. Client
review changed the product direction to a global Admin-managed glossary that
detects matching terms in participant-facing question text and renders inline
definitions.

The global glossary work is larger than a single tooltip pass, so it has been
split into staged follow-up prompts:

* `prompts/prompt_28.txt` — Global Glossary Data Model And Admin Foundation
* `prompts/prompt_29.txt` — Dictionary-Assisted Glossary Definitions
* `prompts/prompt_30.txt` — Participant Inline Glossary Rendering

Keep this file as a historical marker so future phase planning does not
accidentally revive the older per-question tooltip scope.

---

# Superseded Scope

The old scope would have:

* reused `survey_questions.help_text`
* shown an info icon beside a question prompt
* displayed manually entered per-question help text in a tooltip/popover

That is no longer the intended direction for this feature.

---

# Current Product Direction

Admins should be able to maintain a global glossary. Each glossary entry should
have a canonical term, optional match strings/aliases, and an Admin-approved
definition. Participant-facing survey text should automatically identify
matching glossary strings and render accessible inline definition affordances.

The definition popup should be generated at render time from the latest
participant-safe glossary payload, not manually embedded into each question.

---

# Notes For Future Implementation

* Do not store glossary terms as answer options.
* Do not couple glossary definitions to response validation, skip logic,
  hidden tags, reporting, or CSV exports.
* Do not expose Admin-only glossary metadata to participant APIs.
* Verify the current dictionary/Webster provider documentation, terms, and API
  key requirements before implementing the dictionary-assist phase.
