# Scope

Implement:

* User profile/settings route in the user portal.
* Minimal demographic fields: organization or department, job title or role,
  and location or region.
* Relational storage for profile metadata, either as columns or a user profile
  table, avoiding JSON for core profile data.
* Authenticated current-user profile read/update endpoint.
* Current-user survey statistics: available, in progress, completed, last
  activity, and completion rate.
* Shared profile and survey-stat types.

## Non-Goals

Do not implement:

* admin profile viewing or editing
* admin password reset initiation
* CRM-style account management
* speculative demographics beyond the approved minimal fields
* public profile pages
* profile photos or file uploads

## Assumptions

* Standard users can only read and update their own profile.
* Email, first name, and last name remain part of the auth user model unless a
  small update is clearly needed for the profile screen.
* Profile stats are derived from existing survey/attempt data.

## Open Questions

* Are any demographic fields required? If unanswered, keep all new demographic
  fields optional.

---

# Acceptance Criteria

The phase is ready for review when:

* User can open profile/settings from navigation.
* User can save demographic fields and see them after reload.
* User survey statistics render on the profile/settings page.
* API enforces current-user-only access.
* Admin-specific user tools are not implemented in this phase.
