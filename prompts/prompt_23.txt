# Received Feedback

On the survey question page, replace numeric progress indicators with an
estimated time remaining display. The survey-taking interface should no longer
show "Question X of Y" or "Page X of Y" labels. Instead, display an estimated
time remaining value based on the number and type of remaining questions/pages.

---

# Intake Classification

* participant-facing UX feature
* shared/client helper logic
* no expected database or backend API change

Expected impact:

* participant runner display
* time estimate helper
* tests for estimate calculation and path handling

---

# Scope

Implement:

* Remove participant-facing "Page X of Y" and "Question X of Y" labels from the
  survey-taking interface.
* Add a simple reusable estimate helper using remaining visible path/questions.
* Use configurable-in-code averages by question type with a fallback average.
* Respect branching/page-jump/skip behavior by using the existing resolved page
  path.
* Display copy such as "About 3 min remaining" or "Less than 1 min remaining".
* Keep the visual progress bar if it remains useful, but avoid numeric
  page/question copy.

## Non-Goals

Do not implement:

* survey navigation changes
* server-owned estimates unless explicitly required
* changes to admin pagination labels
* removal of dashboard/admin "Page X of Y" pagination
* analytics based on actual completion time

## Assumptions

* Estimated time can be approximate.
* The current page's unanswered visible questions count as remaining.
* Question type averages can live in frontend/shared code rather than
  environment config for this foundation phase.

## Open Questions

* Should completed/review states show total time spent? If unanswered, do not
  add elapsed-time tracking in this phase.

---

# Acceptance Criteria

The phase is ready for review when:

* Participant runner shows estimated time remaining instead of numeric progress
  labels.
* Straight survey estimates decrease as the participant advances.
* Branching/page-jump/skip behavior updates the estimate after path changes.
* Admin pagination labels remain unchanged.
* Survey navigation and completion behavior are unchanged.
