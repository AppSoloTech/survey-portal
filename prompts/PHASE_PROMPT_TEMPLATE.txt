# Intake Classification

Before writing code, classify each item as one of:

* bug or regression
* security, privacy, or data-visibility risk
* broken workflow
* feature request
* UX polish
* reporting or data question
* deployment/setup issue
* unclear item needing confirmation

Call out whether any item changes:

* database schema or migrations
* API contracts
* participant-facing survey behavior
* hidden-tag visibility
* authentication or authorization
* deployment or secret handling
* validation/test expectations

Ask for clarification only when a safe implementation cannot be inferred.
