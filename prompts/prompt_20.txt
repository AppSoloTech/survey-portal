# Scope

Implement:

* Admin user detail endpoint with profile metadata and survey statistics.
* Admin user detail view linked from the existing admin users table.
* Admin-triggered password reset initiation through the Phase 18 reset service.
* User-facing notification via email service, not admin-visible tokens.
* Clear admin UI feedback for reset initiation.

## Non-Goals

Do not implement:

* showing reset tokens, password hashes, or reset links to admins
* account impersonation
* admin editing of demographics unless explicitly needed and approved
* broad CRM/account management workflows
* bulk user actions

## Assumptions

* Admin user detail remains read-only for profile demographics in this phase.
* Reset initiation sends the same type of reset email as the user-requested
  flow.
* Existing self-role-change protections remain intact.

## Open Questions

* Should admins be allowed to reset their own password from admin tools, or only
  through profile/settings? If unanswered, allow the same safe reset initiation
  but never expose the token.

---

# Acceptance Criteria

The phase is ready for review when:

* Admin can open a user detail view from user management.
* Detail view shows safe profile metadata and survey statistics.
* Admin can initiate a password reset for a user without seeing token contents.
* Standard users and unauthenticated requests receive 403/401 as appropriate.
* Password hashes and reset secrets are never returned.
